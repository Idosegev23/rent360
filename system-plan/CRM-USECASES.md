# CRM use-cases לתיווך השכרה — מותאם ל-Rent360

מה ה-CRM צריך לעשות בשביל סוכנות השכרה (קריות/חיפה, וואטסאפ-first, צוות קטן). ✅ = קיים · ➕ = להוסיף.
הרעיון: לא "מאגר" — אלא מערכת שדוחפת את העסקה שלב-אחר-שלב ולא נותנת לכלום ליפול.

## 1. שני ה-Pipelines (עמוד השדרה)
**Pipeline בעלי דירות:** מודעה חדשה → נוצר קשר → בשיחה → אישור תיווך → בשיווק → הותאם שוכר → צפייה → **נסגר**.
**Pipeline שוכרים:** פנייה → תושאל/מטוייב → הותאם → צפייה → הגיש מועמדות → **חתם**.
- ✅ יש את כל הישויות והמעברים (properties→approved→tenancies; renters→matches).
- ➕ **תצוגת Kanban** עם השלבים + גרירה, ו-**SLA לכל שלב** (כמה זמן תקוע → פעולה אוטומטית). זה ה-CRM-move הקלאסי שחסר.

## 2. Speed-to-lead (להגיב מיד, לא לפספס)
- ✅ הבוט עונה מיידית 24/7 + handoff.
- ➕ **חוק "ליד ללא מענה"**: אם אין תגובה X שעות → משימה/תזכורת אוטומטית לסוכן. (היום יש callback ידני; להפוך לאוטומטי.)

## 3. ניהול צפיות (Viewings) — הלב של תיווך
- ✅ פגישות + תזכורות וואטסאפ + קישור לשיחה/נכס.
- ➕ **צפייה דו-צדדית**: תזכורת גם לשוכר וגם לבעל הדירה; **מעקב no-show**; **לכידת פידבק אחרי צפייה** ("התאים? למה לא?") → מזין את ההתאמה הבאה. זה use-case ענק שכרגע חסר.

## 4. נורת'רינג / רצפי מעקב (Cadences)
- ✅ תזכורות callback + recheck נכס שנה אחרי.
- ➕ **רצפים אוטומטיים**: שוכר שלא הגיב → נדנוד יום 2/5; בעל דירה ב"מתלבט" → מעקב ממוקד; מנוע if-this-then-that שהצוות מגדיר.

## 5. ה"קצה" של Rent360 (הבידול שלנו = use-cases ייחודיים)
- ✅ **וטינג שוכר** (תלושים/ערבים/צ׳ק ביטחון) + 3 פחדי בעל-הדירה בבוט.
- ➕ **דייר חלופי / השכרה-חוזרת**: כשדייר עוזב — הנכס חוזר אוטומטית ל-pipeline ("re-rent"), והשוכרים המתאימים מתויגים מיד.
- ➕ **Tenant Score → תג-אמון לשוכר**: סקרי שביעות-רצון (התחלנו ב-tenancies) שמזינים דירוג שוכר ומשפרים התאמות עתידיות.
- ➕ **תיק בעל-דירה (Owner portfolio)**: בעל דירה עם כמה מודעות (ראינו אחת עם 40) = כרטיס-בעלים אחד, בלי לספמם, עם נרצור להשכרות הבאות.

## 6. מסמכים, עמלות, ציות
- ➕ **כספת מסמכים פר-עסקה**: הסכם-תיווך חתום, ת"ז/תלושים, ליווי חתימת חוזה (אנחנו לא מנסחים — מלווים), ביטחונות.
- ➕ **מעקב עמלות**: צפוי מול גבוי, פר-סוכן, פר-עסקה (המודל: חודש + מע"מ — כבר בבוט).

## 7. ייחוס מקור + ROI + ביצועי-צוות
- ➕ **מקור-ליד** (יד2/הפניה/אורגני) → המרה לפי מקור.
- ➕ **דשבורד סוכן**: זמן-להשכרה, צפיות→סגירות, עומס פתוח. (יש כבר "הבוקר שלי"; להרחיב לדוחות.)
- ➕ **הקצאת לידים** (round-robin / לפי אזור) במקום ידני.

## 8. היגיינת-דאטה
- ✅ dedup לפי טלפון בתור השליחה.
- ➕ **איחוד אנשי-קשר** (בעלים/שוכרים/ספקים) לגרף-אנשים אחד; מניעת כפילויות חוצת-מערכת.

---

## מה לבנות קודם (לפי ROI לתיווך השכרה)
1. **צפיות + פידבק-אחרי-צפייה** (#3) — הכי קרוב לכסף; ממנף את הפגישות שכבר נבנו.
2. **Kanban pipeline + SLA** (#1) — הופך את המערכת מ"רשימות" ל"מנוע עסקה".
3. **Speed-to-lead אוטומטי + רצפי-מעקב** (#2,#4) — שאף ליד לא נופל.
4. **Owner portfolio + re-rent** (#5) — ייחודי ל-Rent, ערך חוזר.
5. **מסמכים + עמלות** (#6) — סוגר את העסקה תפעולית.
6. **ROI/דוחות + הקצאה** (#7) — שכבת-ניהול.

עיקרון: כל use-case חדש = עוד טריגר שמזין את **Action Center / משימות / תזכורות** — המערכת עובדת בשבילך, לא להפך.
